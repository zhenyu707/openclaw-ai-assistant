## 1. Incident Classification
- **Critical:** Business operations halted, safety risk, data breach
- **High:** Major service degradation, significant impact
- **Medium:** Partial impact, workaround available
- **Low:** Minor issue, no immediate business impact

## 2. Response Timeline
- Critical: Acknowledge [15]min, Update [30]min, Resolve [4]hr
- High: Acknowledge [30]min, Update [1]hr, Resolve [8]hr
- Medium: Acknowledge [2]hr, Update [4]hr, Resolve [2] days
- Low: Acknowledge [8]hr, Resolve [5] days

## 3. Incident Lifecycle
1. **Detect** — reported by employee, monitoring, or vendor
2. **Log** — create incident record with severity and description
3. **Assign** — route to appropriate team/individual
4. **Investigate** — identify root cause
5. **Resolve** — implement fix and verify
6. **Review** — post-incident review within [48] hours

## 4. Escalation Path
- Unresolved after SLA: → Operations Manager
- Safety/security incident: → immediate escalation to Ops Manager + Legal
- Data breach: → immediate escalation to Legal + Security + Ops Manager

## 5. Contact
For incident questions, contact ops@yourcompany.com.
